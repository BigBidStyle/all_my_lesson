Контекст приложения.
1) --> Когда сервер получает запрос от клиента может быть создан контекст приложения, если нет контекстного приложения ,
 то он ее и не создает.
С контекстом приложения связана две глобальные переменные:
    G --> Тут сохраняется пользовательская информация необходимая для обработки запроса (например соединение с базой
    данных), а когда запрос завершит свою работу, он закроет соединение с базой данных. Так же по завершении работы
    запросы, данные из этого объекта автоматически удаляются.

Контекст запроса.
Когда сервер получает запрос от клиента всегда создается контекст запроса. Который и является обработчиком запроса.
Requests --> cодержит данные с текущим запросом, включая текущий url. Она доступна в пределах текущиго запроса.

Session --> Словарь в котором можно сохранять данные в течение запроса, но уникально для каждого источника.
